## Scope

Adds a lightweight, self-contained web dashboard that can be optionally served alongside the existing MCP server or standalone via `mapx ui`. The dashboard provides visual exploration of the codebase graph, a live tool-call log, a metrics panel, a symbol explorer, and an interactive context viewer. The server path uses Node.js built-ins only (no Express or other runtime framework). The client bundle (< 200 KB gzipped) uses Cytoscape.js for graph rendering, uPlot for charts, and plain TypeScript with no UI framework.

Key deliverables:
- `src/ui-server.ts` — HTTP server (static assets + REST API + SSE)
- `src/ui-events.ts` — shared EventEmitter between MCP interceptor and dashboard SSE stream
- `src/ui/` — client source compiled to `dist/ui/` by `scripts/build-ui.ts`
- `mapx ui` CLI command with `--port`, `--host`, `--no-open`, `--token`, `--dir` flags
- `mapx serve --ui` flag to run MCP server and dashboard in a single process
- MCP tool call interception shim (wraps existing tool handlers to emit events for the Tool Call Log)
- `dist/ui/` shipped in the npm package (added to `files` array in `package.json`)
- `MAPX_NO_UI=1` env flag to skip client build in CI

## Phases

### Phase 1: Server scaffolding + static serving

- `src/ui-events.ts` — `UiEventBus` singleton
- `src/ui-server.ts` — HTTP server serving `dist/ui/` static assets + `/api/status` endpoint
- `mapx ui` command wired into `src/cli.ts` + `src/main.ts`
- `scripts/build-ui.ts` — esbuild pipeline producing `dist/ui/main.js` + copy of `index.html`/`styles.css`
- Minimal `src/ui/index.html` + `src/ui/main.ts` that render the status panel

### Phase 2: Graph Explorer + Symbol Explorer

- `/api/graph` endpoint (paginated file nodes + edges)
- `/api/symbols` + `/api/symbol/:name` endpoints
- Cytoscape.js force-directed graph with node size/colour encoding
- Side-panel on node/edge click
- Symbol table with search and sort
- Language and edge-type filters

### Phase 3: Tool Call Log (SSE)

- `wrapTool()` interceptor in `src/mcp.ts`
- `GET /events` SSE endpoint
- `src/ui/components/tool-log.ts` live scrolling log
- `mapx serve --ui` combined-server mode

### Phase 4: Metrics Panel + Context Viewer

- `/api/metrics` endpoint
- PageRank + coupling + edge-type charts using uPlot
- `/api/context` endpoint (proxies F19 ContextBuilder if available; graceful fallback if not)
- Context Viewer component with task input and ranked file list
- `/api/routes` endpoint (when F21–F26 data is present)

## Blockers / notes

Depends on I07 (F13) for the npm packaging infrastructure (`files` array, `dist/` layout). All other dependencies (F02, F14–F16, F18, F19, F21–F26) are optional — the dashboard degrades gracefully when their data is absent. The client build can be skipped in environments that don't need the UI by setting `MAPX_NO_UI=1`.
