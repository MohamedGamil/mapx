## Objective

Fix the critical gaps in the PHP parser that produce incorrect or incomplete results even for plain PHP projects — before any Laravel-specific improvements are added.

Three features are bundled here because they are independent, low-risk, and together they raise the baseline quality from "broken for namespaced PHP" to "reliable for modern PHP 8.x".

---

## Feature Bundle

| Feature | Summary |
|---------|---------|
| F05 | Fix `use` import capture — extract full qualified names, not just the last segment |
| F06 | Extract type-hint dependency edges — constructor injection, method params, return types |
| F10 | Laravel-aware noise reduction — exclude migrations, seeds, factories, blade templates |

F10 is included in I04 (not I05) because it is fully independent of Laravel-specific parser logic, and the exclusions dramatically improve scan quality for anyone using this parser on a Laravel codebase.

---

## F05 Summary: FQN Resolution

**Problem**: `use App\Services\UserService;` is currently captured as `name = "UserService"` (last segment only), producing edges to `UserService` instead of `App\Services\UserService`.

**Fix**: Change `queries/php/references.scm` to capture `(qualified_name)` instead of `(name)` for `namespace_use_clause` nodes. Build a `UseImportTable = Map<shortName, fqn>` during parsing. Apply in all subsequent reference resolutions.

**Risk**: Low. Tree-sitter query change only; no structural changes to `php.ts`.

---

## F06 Summary: Type-Hint Edges

**Problem**: `__construct(private UserService $service)` creates a real dependency on `UserService`, but this edge is not emitted.

**Fix**: Add tree-sitter queries to `references.scm` for:
- Constructor parameter types (standard and promoted)
- Method parameter types
- Return type declarations
- Typed properties

Filter out PHP scalar types. Resolve all captured type names via the F05 use-import table.

**Risk**: Low-Medium. New query patterns and a type resolution helper function.

---

## F10 Summary: Noise Reduction

**Problem**: Migrations, seeders, factories, `bootstrap/cache`, and Blade templates produce hundreds of false edges and inflate symbol/file counts.

**Fix**: Add `LARAVEL_DEFAULT_EXCLUDES` patterns. Detect Laravel projects in `mapx init`. Classify and suppress edges from migration/seeder/factory files. Exclude `**/*.blade.php`.

**Risk**: Low. Config-level change; no parser logic changes. Defaults only apply when Laravel is detected.

## Acceptance Criteria (I04 complete when all pass)

- [ ] `use App\Services\UserService;` → FQN `App\Services\UserService` in edges
- [ ] Aliased use `use App\Services\UserService as US;` → `US` resolves to `App\Services\UserService`
- [ ] Grouped use `use App\Services\{UserService, OrderService};` → both FQNs captured
- [ ] Constructor injection `__construct(UserService $svc)` → `param_type` edge to FQN
- [ ] Promoted property `__construct(private readonly UserService $svc)` → `param_type` edge
- [ ] Return type `function handle(): Response` → `return_type` edge to FQN
- [ ] PHP scalars (`string`, `int`, `bool`, `array`, `void`, `mixed`, `never`, `float`) not emitted as edges
- [ ] `mapx init` on a Laravel project prompts with exclusion suggestions
- [ ] Migration files produce zero edges when scanned
- [ ] `**/*.blade.php` excluded from PHP parser
- [ ] `bootstrap/cache/**` files return empty ParseResult
- [ ] TypeScript type-check: `npx tsc --noEmit` passes with 0 errors
- [ ] All existing tests pass (no regressions)
