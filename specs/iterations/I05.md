## Objective

Add parser support for the three structural patterns that define the architecture of every Laravel application:

1. **Eloquent models** ‚Äî the data layer; relationship methods define the domain model shape
2. **Routes** ‚Äî the HTTP entry points; bindings to controllers are the application surface area
3. **Service providers** ‚Äî the IoC wiring; bindings from interface to concrete establish the dependency inversion topology

Together these three features transform mapx from a generic PHP symbol graph into a tool that understands *Laravel application architecture*.

## F07 Summary: Eloquent Relationships

**What it adds**: A new `relation` edge type from model class to related model class. The relationship method name (`hasMany`, `belongsTo`, etc.) and type (`one-to-many`, `many-to-one`, etc.) are stored as edge metadata.

**Requires F05**: The `::class` constant in `$this->hasMany(Post::class)` must be resolved to `App\Models\Post` via the use-import table.

**New metadata**: `laravelRole = 'model'` set on Eloquent model classes.

**Graph impact**: For a typical 30-model application, this adds ~60‚Äì120 `relation` edges connecting the data model ‚Äî an entirely new layer of graph structure that was previously invisible.

---

## F08 Summary: Route-to-Controller Binding

**What it adds**: A new `route` edge type from route files to controller classes/methods. HTTP verb and URI stored as metadata. A new `middleware` edge type for `->middleware()` chaining.

**Requires F05**: Array-form `[UserController::class, 'index']` must be resolved to the FQN.

**New metadata**: `laravelRole = 'route_file'` on route files, `laravelRole = 'controller'` on controller classes.

**Graph impact**: Route files become meaningful nodes that connect the HTTP surface area to the controller layer. Previously they were near-useless (dozens of `call ‚Üí Route` edges). After F08 they show the full routing topology.

---

## F09 Summary: Service Provider Bindings

**What it adds**: A new `binding` edge type from abstract (interface/alias) to concrete (implementation). Singleton vs. transient vs. scoped vs. alias stored as metadata.

**Requires F05**: `UserRepositoryInterface::class` and `UserRepository::class` must be resolved to FQNs.

**New metadata**: `laravelRole = 'service_provider'` on service provider classes. `serviceProviderPhase = 'register' | 'boot'` on edges.

**Graph impact**: Interfaces used as type hints (F06) that previously had no concrete target now have an explicit `binding` edge to their implementation. The IoC container topology becomes visible.

## Acceptance Criteria (I05 complete when all pass)

### F07 ‚Äî Eloquent

- [ ] `$this->hasMany(Post::class)` ‚Üí `relation` edge, `User ‚Üí App\Models\Post`, `one-to-many`
- [ ] `$this->belongsTo(Team::class)` ‚Üí `relation` edge, `many-to-one`
- [ ] `$this->belongsToMany(Role::class)` ‚Üí `relation` edge, `many-to-many`
- [ ] `$this->morphMany(Comment::class, 'commentable')` ‚Üí `relation` edge
- [ ] `laravelRole = 'model'` set on Eloquent model classes
- [ ] `relation` edges rendered distinctly (colour) in DOT/SVG exporter
- [ ] LLM exporter shows relationship edges in model section

### F08 ‚Äî Routes

- [ ] `Route::get('/users', [UserController::class, 'index'])` ‚Üí `route` edge, `verified`
- [ ] `Route::post('/users', 'UserController@store')` ‚Üí `route` edge, `inferred`
- [ ] `Route::resource('orders', OrderController::class)` ‚Üí `route` edge, `resourceType: 'resource'`
- [ ] `->middleware('auth')` ‚Üí `middleware` edge
- [ ] Route group prefix and middleware inherited by children
- [ ] `laravelRole = 'route_file'` on route files
- [ ] `laravelRole = 'controller'` on controller classes
- [ ] LLM exporter groups route edges by controller class

### F09 ‚Äî Service Providers

- [ ] `$this->app->bind(Interface::class, Concrete::class)` ‚Üí `binding` edge
- [ ] `$this->app->singleton(Service::class, ...)` ‚Üí `binding`, `bindingType = 'singleton'`
- [ ] Closure-form concrete ‚Üí `binding` edge with `concreteClass: null`
- [ ] `$this->app->alias(Interface::class, 'alias')` ‚Üí `binding`, `bindingType = 'alias'`
- [ ] `laravelRole = 'service_provider'` on service provider classes
- [ ] `serviceProviderPhase` set correctly for `register()` / `boot()`

### Common

- [ ] TypeScript type-check: `npx tsc --noEmit` passes with 0 errors
- [ ] All I04 acceptance criteria continue to pass (no regressions)
